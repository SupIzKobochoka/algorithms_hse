string - строка, где ищем подстроку
substring - подстрока, которую ищем

Суффикс - непрерывная подстрока с конца строки любой длинны
Префикс - непрерывная подстрока с начала строки любой длинны

Для начала в алгоритме Knuth–Morris–Pratt, строится последовательность соответствующая substring,
где для каждого элемента указывается размер наибольшего суффикса, такого,
что для этого суффикса существует префикс левее текущего элемента, равный этому суффиксу.
Либо 0, если такого нет
Например 'abcabc': [0, 0, 0, 1, 2, 3] (prefix)

Далее проходим по substring и string. 
В случае равенства элементов сравниваем следующие элементы, 
если элементы не равны, начинаем сравнивать текущий символ string 
с substring[prefix[текущий_индекс_substring]], если индекс указывает на 0, 
то сдвигаемся на следующий символ string и сравниваем с самого начала substring

Делается это для того, чтобы проверять лишь действительно возможные совпадения пар.



Сложность:
n - длинна строки
m - длинна подстроки

Подсчет prefix - O(m) (там алгоритм есть эффективный)
Далее в алгоритме мы по сути последовательно передвигаем указатели для string и substring, 
где в худшем cлучае после прохода k символов, нам нужно будет вернуть указатель для substring на k символов назад.
Т.е. получаем в итоге в любом случае O(n)
Итого O(m+n)
